# Contributing

Thank you for your interest in contributing to this project!

## How to Contribute

1. **Fork the repository** and create your branch from `main`
2. **Make your changes** and ensure they follow the existing code style
3. **Test your changes** thoroughly
4. **Submit a pull request** with a clear description of your changes

## Code Style

- CloudFormation templates: Use YAML format with consistent indentation (2 spaces)
- Python scripts: Follow PEP 8 guidelines
- PowerShell scripts: Use approved verbs and consistent naming

## Reporting Issues

When reporting issues, please include:
- A clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- AWS region and relevant resource information (without sensitive data)

## Pull Request Process

1. Update documentation if your changes affect usage
2. Ensure all CloudFormation templates are valid (`aws cloudformation validate-template`)
3. Test deployments in a non-production environment
4. Update the README.md if adding new features

## Questions?

Open an issue for any questions about contributing.
